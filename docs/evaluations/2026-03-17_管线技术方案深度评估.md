## 一、管线一：文章提取（URL → 笔记）

### 技术方案对比

| 方案 | 体积 | 中文支持 | 反爬能力 | 图片提取 | 推荐 |
|------|------|---------|---------|---------|------|
| trafilatura | 21MB | ✅ | ❌ 微信拦 | ⚠️ 有限 | ✅ 通用网页首选 |
| Crawl4AI | ~80MB | ✅ | ✅ JS渲染 | ✅ | ❌ 太重 |
| Playwright | ~50MB+Chrome | ✅ | ✅ 最强 | ✅ | ✅ 微信兜底 |
| Jina Reader API | 0MB | ✅ | ✅ 云端 | ⚠️ | ✅ 第二层fallback |

### 推荐：多层 Fallback 架构

```
输入 URL
  │
  ▼
trafilatura (< 1s, 通用网页)     ← 第一层：快且轻
  │ 失败
  ▼
Jina Reader API (3-5s, 云端)     ← 第二层：处理JS渲染页
  │ 失败  
  ▼
Playwright headless Chrome       ← 第三层：终极兜底（微信等）
  │ 失败
  ▼
镜像站搜索 (53ai.com/ofweek等)   ← 第四层：微信专用绕过
```

### 图片处理策略
- **本地 Markdown**：图片下载到 `images/` 子目录，相对路径引用
- **飞书文档**：图片上传到云盘，Markdown 中引用（飞书导入时保留位置）

## 二、管线二：OCR（图片 → Markdown）

### 引擎对比

| 引擎 | 体积 | 精度 | 中文 | 表格 | 公式 | API状态 |
|------|------|------|------|------|------|---------|
| PaddleOCR 3.4 | 500MB | 94.5% | ✅ 最强 | ✅ | ✅ | ⚠️ API变更 |
| rapidocr-onnxruntime | 225MB | 92% | ✅ | ✅ | ⚠️ | ✅ 稳定 |
| tesseract 5 | 30MB | 85% | ⚠️ | ❌ | ❌ | ✅ 稳定 |
| 百度OCR API | 0MB | 95%+ | ✅ | ✅ | ✅ | ✅ 需key |

### PaddleOCR 3.4 API 问题
```
错误：Unknown argument: show_log
错误：PaddleOCR.predict() got an unexpected keyword argument 'cls'
错误：Unimplemented ConvertPirAttribute2RuntimeAttribute
```

**解决方案**：
- 移除 `show_log` 和 `cls` 参数
- 使用 `engine.predict(image)` 而非 `engine.ocr(image)`
- 检查 oneDNN 后端兼容性（设置 `FLAGS_use_mkldnn=0`）

### 推荐方案

**短期（快速可用）：rapidocr-onnxruntime**
- 体积 225MB，精度 92%
- API 稳定，开箱即用
- 安装：`pip install rapidocr-onnxruntime`

**长期（精度优先）：PaddleOCR**
- 解决 API 兼容问题后切换
- 精度最高（94.5%）
- 支持表格/公式识别

**轻量备选：tesseract + LLM 校错**
- 体积仅 30MB
- 精度 85%，但 LLM 校错后可提升到 ~92%
- 适合分层安装的 Core 方案

### 多图处理策略
```
图片目录 → 逐张 OCR → 合并文本 → LLM 校错 → 输出 Markdown
```
- 单图分别 OCR，保留每张图的位置信息
- 合并时按文件名排序
- LLM 校错一次性处理全部文本（减少 API 调用）

## 三、管线三：视频（链接 → 笔记）

### BiliNote 架构分析

**技术栈**：React + FastAPI + yt-dlp + Whisper + LLM

**核心流程**：
1. yt-dlp 下载视频字幕（优先）或音频
2. Whisper 转录音频为文本
3. LLM 生成结构化笔记
4. ffmpeg 提取关键帧截图
5. 组装 Markdown（含截图+时间戳跳转）

**关键特性**：
- 支持 B站/YouTube/抖音/快手/本地视频
- 可选插入截图（自动截取关键帧）
- 可选内容跳转链接（关联原视频）
- 笔记风格选择（学术/口语/重点）
- 任务历史记录

**可复用组件**：
- 字幕提取逻辑（yt-dlp 调用方式）
- 截图提取策略（ffmpeg 命令）
- 笔记模板设计

### 视频管线方案

| 平台 | 字幕来源 | 工具 | 备注 |
|------|---------|------|------|
| B站 | 官方字幕/弹幕 | yt-dlp | 需 cookies |
| YouTube | 自动字幕/CC | yt-dlp | 需代理 |
| 抖音 | 无字幕为主 | Whisper | 音频转录 |
| 通用 | 音频流 | Whisper | 降级方案 |

### 关键帧截图策略
```python
# ffmpeg 提取关键帧（场景变化检测）
ffmpeg -i video.mp4 -vf "select=gt(scene,0.3)" -vsync vfr frame_%03d.png

# 按时间间隔截图
ffmpeg -i video.mp4 -vf "fps=1/60" frame_%03d.png  # 每分钟一帧
```

### 笔记模板

## 四、管线四：智能路由

### 路由策略

```python
def detect_input_type(input_str):
    # 1. URL 检测
    if input_str.startswith(('http://', 'https://')):
        domain = extract_domain(input_str)
        if domain in VIDEO_DOMAINS:
            return 'video_url'
        elif domain in WECHAT_DOMAINS:
            return 'wechat_url'  # 特殊处理
        else:
            return 'url'
    
    # 2. 文件/目录检测
    path = Path(input_str)
    if path.is_dir():
        return 'directory'
    elif path.suffix.lower() in IMAGE_EXTS:
        return 'image'
    elif path.suffix.lower() in VIDEO_EXTS:
        return 'video_file'
    
    return 'unknown'
```

### 混合输入处理
- 支持多种输入：`kb img1.png url1 url2 video.mp4`
- 按类型分组，分别路由到对应管线
- 输出合并为一个文档或分别输出
