> 版本：v4.0 | 日期：2026-03-17 | 状态：待 Boss 确认

## 1. TL;DR

**一个 CLI 工具：`kb <材料>` → 得到 Markdown 笔记。零配置，开源，模块化安装。**

- 🎯 知识囤积者：收藏夹 500+ 链接没读，截图 200+ 没整理
- 🔧 智能路由 → 内容提取 → AI 旁注 → Markdown 输出
- 🔌 模块化安装（核心 50MB）+ 本地/API 双通道
- 🤖 LLM 走 OpenClaw/API，不做本地模型

---

## 2. 产品是什么

### 一句话定位

**面向个人的零配置知识摄入引擎——扔进材料，输出笔记。**

### 核心理念

1. **原文优先**：默认保留完整信息，AI 加旁注帮用户定位，不替用户删内容
2. **智能路由**：用户不需要指定类型，`kb <anything>` 自动识别
3. **模块化**：核心 50MB 起步，按需安装组件，本地/API 自由切换

### 不是什么

- ❌ 不是笔记编辑器（Obsidian/Logseq 已经做得很好）
- ❌ 不是收藏夹（Cubox/简悦做收藏）
- ❌ 不是格式转换器（MarkItDown 做转换）
- ❌ 不是 RAG 工具（Crawl4AI 做 RAG）

## 3. 用户

### 用户画像：知识囤积者

| 维度 | 描述 |
|------|------|
| 是谁 | 技术人、研究者、内容创作者 |
| 工具链 | Obsidian/Logseq/Typora + 微信/B站/飞书 |
| 行为模式 | 看到好内容 → 收藏 → "有空整理" → 没空 |
| 痛苦 | 收藏夹爆满、截图积灰、视频看完就忘 |

### 用户故事

**故事 1：截图整理（李姐，产品经理）**
```
kb ./screenshots/
```
→ 5 秒得到结构化 Markdown。之前手动 40 分钟的活儿。

**故事 2：文章抢救（小明，工程师）**
```
kb "https://mp.weixin.qq.com/s/xxxxx"
```
→ 完整正文 + AI 旁注（摘要/要点/数据/待读）。两周后文章被删，笔记还在。

**故事 3：视频笔记（老王，研究者）**
```
kb "https://bilibili.com/video/BVxxx"
```
→ 字幕 + 关键帧截图 + 时间戳笔记。（参考百度网盘 AI 笔记形态）

**故事 4：飞书即用（Boss）**
Boss 飞书发截图给灵虾 → 调用 kb → 返回结构化文字 → 写入飞书文档。全程对话。

**故事 5：批量归档**
```bash
kb batch links.txt --output ~/notes/march/
```
20 个链接，10 分钟全部变成笔记。

### 成功体感

> "遇到有价值内容，第一反应是 `kb` 一下而不是收藏。因为**收藏=忘记，kb=笔记已在库里**。"

## 4. 做什么

### v1.0 功能清单

| 能力 | 输入 | 输出 | 安装 | 优先级 |
|------|------|------|------|--------|
| 文章→笔记 | URL | 原文+AI旁注 | 核心（必装） | P0 |
| 图片→文档 | 图片/目录 | 结构化Markdown | `kb-tool[ocr]` | P0 |
| 视频→笔记 | 视频URL | 字幕+截图+时间戳笔记 | `kb-tool[video]` | P0 |
| AI旁注 | 任意内容 | 摘要/要点/数据/待读 | `kb-tool[ai]` | P0 |
| 智能路由 | 任意输入 | 自动识别类型 | 核心（必装） | P0 |
| CLI | 命令行 | 一行命令搞定 | 核心（必装） | P0 |
| OpenClaw集成 | Agent调用 | 飞书对话即用 | `kb-tool[openclaw]` | P0 |
| 批量处理 | 文件/链接列表 | 批量Markdown | 核心 | P1 |

### v1.0 不做什么

| 不做 | 理由 | 何时做 |
|------|------|--------|
| 编辑器 | Obsidian 已做好 | 永远不做 |
| 云同步 | 用户自己搞 | 永远不做 |
| 团队协作 | 个人工具 | v3 考虑 |
| 知识图谱 | 下游工具的活儿 | 永远不做 |
| Web UI | CLI 先验证价值 | v2 考虑 |
| 全文搜索 | ripgrep 更好 | 永远不做 |
| PDF 解析 | v1 只做图片 | v1.1 增量 |
| Readwise导入 | 集成是v2的事 | v2 考虑 |
| 本地LLM | 7B质量不达标，大模型用户跑不动 | 不做 |

### 提炼策略：原文+AI 旁注

**默认模式（fidelity）：原文完整 + AI 旁注**

```
# 原文标题

> 📝 摘要：这篇文章介绍了...（3句话）
> 🏷️ 标签：AI, 知识管理, CLI

[原文完整内容，结构化Markdown]

> 💡 要点：这段核心论点是...
> 📊 数据：关键数字 42%
> 🔗 链接：相关 /wiki/xxx
> 📖 待读：这段值得回头精读
```

**可选模式：**
- `--mode=fidelity` — 原文+旁注（默认）
- `--mode=concise` — 去水提炼（用户主动选）
- `--mode=raw` — 纯原文，不加工

## 5. 怎么做

### 模块化架构

```bash
pip install kb-tool           # 核心（<50MB）：trafilatura + 路由 + Markdown
pip install kb-tool[ocr]      # +OCR（~500MB）：PaddleOCR
pip install kb-tool[video]    # +视频（~60MB）：yt-dlp + ffmpeg
pip install kb-tool[ai]       # +AI旁注：LLM API客户端
pip install kb-tool[whisper]  # +本地转录（1-6GB）：Whisper
pip install kb-tool[all]      # 全部（不含本地LLM）
```

### 组件：本地/API 双通道

| 组件 | 默认 | 本地方案 | API 方案 | 体积 |
|------|------|---------|---------|------|
| 网页提取 | 本地 | trafilatura | Jina Reader | ~20MB |
| OCR | 本地 | PaddleOCR | 百度/腾讯/Azure | ~500MB |
| LLM 旁注 | API | — | OpenRouter / OpenClaw | ~0 |
| 视频字幕 | 本地 | yt-dlp + Whisper | 讯飞 | ~60MB (不含Whisper) |
| 语音转录 | 可选 | Whisper | Whisper API | 1-6GB |

### 智能路由

```
输入 → URL? → 视频域名? → 视频管线
              → 否 → 文章管线
    → 文件? → 图片后缀? → OCR 管线
    → 无法识别 → 提示手动指定
```

### 与 OpenClaw 的关系

- **LLM 复用 OpenClaw 配置**，零额外成本
- **作为 OpenClaw Skill**，飞书对话直接用
- **同时支持独立 CLI**，非 OpenClaw 用户也适用

## 6. 差异化

### 竞品对比

| 维度 | kb-tool | MarkItDown | Crawl4AI | 百度网盘AI笔记 |
|------|---------|-----------|----------|--------------|
| 智能路由 | ✅ 自动识别 | ❌ | ❌ | ❌ |
| 信息保留 | ✅ 原文+旁注 | ✅ 纯转换 | ✅ 纯转换 | ⚠️ 提炼为主 |
| 多平台视频 | ✅ B站/抖音/YouTube | ⚠️ YouTube | ❌ | ❌ 网盘内 |
| 可编程 | ✅ CLI+API | ✅ | ✅ | ❌ |
| 本地优先 | ✅ | ✅ | ✅ | ❌ |
| 开源 | ✅ MIT | ✅ Apache | ✅ Apache | ❌ |

### 核心优势

1. **唯一"原文+AI旁注"模式** — 不丢信息密度，其他工具要么纯转换要么提炼
2. **视频笔记最完整** — 字幕+关键帧截图+时间戳笔记，B站/抖音无人做
3. **OpenClaw 生态原生** — LLM 复用，飞书即用

### 护城河

| 护城河 | 强度 |
|--------|------|
| 中文站点适配规则 | 🔴 强 |
| AI旁注 Prompt 工程 | 🔴 强 |
| 视频关键帧+笔记管线 | 🔴 强 |
| OpenClaw 生态绑定 | 🔴 强 |
| 先发优势 | 🟡 中 |

## 7. 技术约束

### 技术选型

| 决策 | 选择 | 理由 |
|------|------|------|
| 语言 | Python 3.12 | 生态最好 |
| CLI | Typer | 类型安全，自动帮助 |
| OCR | PaddleOCR 3.0 | 94.5%精度，CPU可用 |
| 网页 | trafilatura | benchmark最优 |
| 视频 | yt-dlp | 1000+站点 |
| 转录 | Whisper | 可选安装 |
| LLM | OpenRouter / OpenClaw | 复用配置 |
| 存储 | Markdown + SQLite | 用户数据可控 |
| License | MIT | 最宽松 |

### 硬件要求

| 配置 | 能跑什么 | 安装体积 |
|------|---------|---------|
| 2核4GB VPS | 核心 + API处理 | <50MB |
| 4核8GB 普通笔记本 | 核心 + OCR + 视频字幕 | ~600MB |
| 8核16GB + GPU | + Whisper本地转录 | ~3GB |
| WSL2 + RTX 4060 | 全本地模式 | ~3GB |

### 资源开销

| 组件 | 安装体积 | 运行内存 | 处理速度 |
|------|---------|---------|---------|
| 核心 | <50MB | ~200MB | 文章 5-15s |
| OCR | ~500MB | ~1GB | 图片 3-10s/张 |
| 视频字幕 | ~60MB | ~100MB | 字幕下载 2-5s |
| Whisper | 1-6GB | 2-6GB | 1-5x 实时 |
| LLM API | ~0 | ~50MB | 旁注 3-8s |

**月成本（日均 20 篇，API 模式）：** ~¥30-50/月
**全本地模式：** ¥0
