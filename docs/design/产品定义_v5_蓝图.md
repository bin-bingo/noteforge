## 架构图

```
┌──────────────────────────────────────────────┐
│           Core（核心指令）                    │

│   API  →  OpenRouter  →  本地模型  →  OpenClaw│

        输入 ──────────→ 处理 ──────────→ 输出

        平台路由：bilibili / 抖音 / 微信 / YouTube
```

## 与 v4 的差异

| 维度 | v4（当前开发） | v5（蓝图） |
|------|---------------|-----------|
| 输入 | 文章/图片/视频 | + 文档/PDF |
| 模型 | 固定 OpenRouter | 降级链：API→OpenRouter→本地→OpenClaw |
| 图片处理 | 无 | 降噪/裁剪/缩放预处理 |
| 输出 | Markdown + 飞书 | + Notion + Obsidian + 其他 |
| 索引 | 无 | Source + Tag + DB + 搜索 |
| 认证 | 无 | Auth 模块 |

## v5 新增模块详解

### 1. 模型降级链

```
用户配置 / 自动选择
    ↓
OpenRouter API（有 key）
    ↓ 失败
其他 API 备选
    ↓ 失败
本地模型（Ollama/已安装）
    ↓ 无
提示用户配置
```

### 2. 文档/PDF 输入

- PDF 文字提取（PyMuPDF）
- PDF OCR（扫描件）
- Word/Excel/PPT（MarkItDown 集成）

### 3. 图片预处理

- 降噪：去除扫描件噪点
- 裁剪：自动检测文档边界
- 缩放：统一尺寸，提升 OCR 准确率

### 4. 多输出平台

| 平台 | 实现方式 |
|------|---------|
| Markdown | 直接写文件（已有） |
| 飞书文档 | API 写入（已有） |
| Notion | Notion API |
| Obsidian | 直接写 vault 目录 |

### 5. 索引与搜索

```
SQLite 数据库：
- source: 原始 URL/文件路径
- tags: 用户标签 + AI 自动标签
- created: 创建时间
- type: article/video/ocr/document
- summary: AI 生成的摘要
- full_text: 全文（用于搜索）
```

搜索功能：
- 按标签筛选
- 按时间范围
- 按类型
- 全文搜索（SQLite FTS5）

## 实现优先级（建议）

| 优先级 | 模块 | 理由 |
|--------|------|------|
| P0 | 核心管线（文章/图片/视频） | v4 正在做 |
| P1 | 模型降级链 | 提高可靠性 |
| P2 | 文档/PDF 输入 | 扩大输入范围 |
| P3 | 多输出（Notion/Obsidian） | 扩大用户群 |
| P4 | 索引与搜索 | 增强检索能力 |
| P5 | 图片预处理 | 精度提升 |
