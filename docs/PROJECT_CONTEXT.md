# 个人知识库构建工具 — 项目上下文

## 背景
Boss 需要一个个人知识库构建工具，自动将多种来源的内容提炼为结构化 Markdown 笔记。

## 核心需求

**产品理念：零配置，提供材料即可获得笔记。** 用户不需要知道用什么 OCR、什么提取引擎，不需要手动配置。扔材料进来，出笔记。

### 1. 多图 OCR + 文档合成
- 来源多样：飞书传图、文档内图片、截图等，用户扔图就行
- 自动识别图片内容，OCR + LLM 校错，合成结构化文档

### 2. 文章链接 → 笔记
- 用户贴个链接，自动提取正文+AI旁注
- **默认"原文+旁注"模式**（保留完整信息密度），用户可选"提炼"模式
- AI 旁注：3句摘要、关键论点标注、数据高亮、相关链接、待读标记

### 3. 多媒体链接 → 学习笔记
- 用户贴个视频链接（抖音、B站、YouTube），自动字幕提取+总结
- 输出：结构化学习笔记

### 4. 交付形式
- CLI 工具，与 OpenClaw 集成
- **智能路由**：用户不需要指定类型，工具自动识别输入是图片/链接/文件
- **零配置**：内置合理默认值，首次运行自动配置

## 技术约束
- **不限技术栈**，选最适合的方案
- **开源项目**，需要考虑社区贡献、文档、license
- 运行环境：WSL2 (Ubuntu), Python 3.12, Node.js 22
- 已有能力：OpenClaw ACP、OpenCode、飞书 API、浏览器自动化
- 已有模型：DeepSeek-OCR-2 (等 GPU)，当前可用 OpenRouter API
- 视频字幕提取：yt-dlp 已装

## 产品原则
1. **产品定义先行**：先把"做什么、不做什么、给谁用"想清楚
2. **前后端不拆分**：当前阶段不需要区分前后端工程师，聚焦产品本身
3. **开源优先**：所有设计决策考虑可维护性、社区友好性
4. **模块化安装**：核心必装（<50MB），按需安装 OCR/视频/AI 等组件
5. **本地/API 双通道**：每个组件支持本地处理和 API 处理，用户自由选择
