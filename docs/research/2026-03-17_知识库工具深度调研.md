# 个人知识库构建工具 — 深度调研报告

> 日期：2026-03-17 | 调研方式：浏览器搜索引擎 + GitHub + 工具文档

---

## 一、调研发现的关键工具

### 1. BiliNote — AI 视频笔记生成器 ⭐5.4k

**地址：** https://github.com/JefferyHcool/BiliNote  
**官网：** https://www.bilinote.app/  
**文档：** https://docs.bilinote.app/

**功能：**
- 支持 B站、YouTube、抖音、本地视频
- 自动字幕提取 → AI 总结 → Markdown 笔记
- **自动截图**（关键帧插入笔记）
- **原片跳转链接**（点击笔记回到视频对应位置）
- 笔记风格选择（学术风/口语风/重点提取）
- 支持多模态视频理解
- 本地 Fast-Whisper 转写 + GPT/DeepSeek/Qwen 总结
- Docker 部署 + Windows exe

**技术栈：** React 前端 + FastAPI 后端 + yt-dlp + Whisper + LLM

**与 kb-tool 的关系：** BiliNote 做视频笔记做得非常好，但只做视频，不做文章和 OCR。

---

### 2. MarkItDown — 微软文档转 Markdown

**地址：** https://github.com/microsoft/markitdown  
**功能：**
- 支持 PDF、Word、PPT、Excel、图片（OCR）、音频（转录）、HTML、CSV、JSON、YouTube URL
- 保留文档结构（标题、列表、表格）
- CLI + Python API + MCP Server
- 插件系统（可扩展）

**局限：** 只做格式转换，不做内容理解和提炼。

---

### 3. Crawl4AI — AI 网页抓取 ⭐50k+

**地址：** https://github.com/unclecode/crawl4ai  
**功能：**
- 网页 → LLM-friendly Markdown
- 支持深度爬取、JavaScript 渲染
- 自动去噪、提取正文
- 支持 Docker 部署

**局限：** 只做网页，不做 OCR/视频。

---

### 4. Obsidian AI 插件生态

**相关插件：**
- **Smart Composer** — AI 写作+总结
- **Text Generator** — AI 内容生成
- **Copilot** — AI 对话查询笔记
- **Media DB** — 从外部源拉取内容
- **Readwise** — 同步阅读高亮

**局限：** 插件各自为政，没有统一的"内容摄入管线"。

---

### 5. WaytoAGI — AI 知识库平台

**地址：** https://www.waytoagi.com/  
**功能：** 收录上千 AI 工具和知识，类似 AI 工具导航+知识库  
**局限：** 是网站导航，不是个人知识库工具。

## 二、竞品格局总结

### 按功能矩阵对比

| 工具 | 文章→笔记 | 视频→笔记 | 图片OCR | 智能路由 | CLI | 开源 |
|------|----------|----------|---------|---------|-----|------|
| **kb-tool (我们)** | ✅ | ✅(规划) | ✅ | ✅ | ✅ | ✅ MIT |
| **BiliNote** | ❌ | ✅ 强项 | ❌ | ❌ | ❌ Web | ✅ MIT |
| **MarkItDown** | ❌ 纯转换 | ⚠️ YouTube | ⚠️ 基础 | ❌ | ✅ | ✅ MIT |
| **Crawl4AI** | ✅ 强项 | ❌ | ❌ | ❌ | ✅ | ✅ Apache |
| **Obsidian插件** | ⚠️ 插件 | ⚠️ 插件 | ⚠️ 插件 | ❌ | ❌ | 混合 |
| **Cubox** | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |
| **简悦** | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |

### 市场空白（我们的机会）

1. **全品类 CLI 入口** — 没有工具同时做文章+视频+图片+智能路由
2. **原文+AI 旁注模式** — 要么纯转换（MarkItDown），要么纯提炼（Cubox），没有"原文+旁注"
3. **OpenClaw 集成** — 没有工具跟 AI Agent 深度集成
4. **零配置体验** — 所有工具都需要配置，没有"pip install 即用"

---

## 三、建议

### 1. 视频管线：借鉴 BiliNote
BiliNote 的截图+时间戳设计做得很好，MIT 开源。可以直接借鉴设计，甚至考虑集成其组件。

### 2. 网页提取：trafilatura 已够用
Crawl4AI 太重（面向 RAG），trafilatura 轻量且准确。微信文章走 Playwright 兜底。

### 3. 文件转换：不需要做
MarkItDown 已经覆盖了 Office→Markdown，我们不需要重复。专注做"内容理解+提炼"。

### 4. 核心差异化保持
我们独特价值：**全品类智能路由 + 原文+AI旁注 + OpenClaw 集成 + 零配置 CLI**
