## 一、产品愿景

**让每个人都能轻松构建个人知识库**

- **目标用户**: 知识囤积者（收藏夹爆满、截图积灰、视频看完就忘）
- **核心价值**: 扔进材料（URL/图片/视频），自动输出结构化 Markdown 笔记
- **产品形态**: CLI 工具 + OpenClaw 生态集成

---

## 二、核心功能（v1.0）

### 2.1 文章链接提炼
**输入**: 微信公众号/CSDN/博客文章 URL  
**输出**: 原文 + AI 旁注（摘要/要点/关键数据/待读标记）  
**技术栈**: trafilatura + Playwright (兜底) + OpenRouter API  
**优先级**: P0 ✅ 已完成

### 2.2 多图 OCR 识别
**输入**: 飞书传图/文档截图/本地图片  
**输出**: 结构化 Markdown 文档（含推理校错）  
**技术栈**: RapidOCR + LLM 校错  
**优先级**: P0 ✅ 已完成

### 2.3 视频链接提炼
**输入**: B 站/抖音/YouTube 视频链接  
**输出**: 字幕 + 时间戳笔记 + 关键帧截图  
**技术栈**: yt-dlp + Whisper + ffmpeg  
**优先级**: P1 ✅ 已完成

### 2.4 批量处理
**输入**: 多个 URL/图片/文件  
**输出**: 批量生成 Markdown 文件  
**技术栈**: CLI batch 命令  
**优先级**: P0 ✅ 已完成

## 三、产品特性

### 3.1 核心理念
- **原文优先**: 默认保留完整信息，AI 旁注帮助定位，不替用户删内容
- **智能路由**: 自动识别输入类型（URL/文件/目录/视频）
- **模块化**: 核心 50MB 起步，按需安装组件
- **零配置**: 开箱即用，默认配置适合大多数场景

### 3.2 安装模式
```bash
# 核心功能（<50MB）
pip install kb-tool

# +OCR 功能（~500MB）
pip install 'kb-tool[ocr]'

# +视频功能（~60MB）
pip install 'kb-tool[video]'

# 完整功能（~3GB）
pip install 'kb-tool[all]'
```

### 3.3 处理模式
| 模式 | 说明 | 适用场景 |
|-----|------|---------|
| `fidelity` (默认) | 原文+AI 旁注 | 深度阅读、学习研究 |
| `concise` | 去水提炼 | 快速浏览、信息筛选 |
| `raw` | 纯原文 | 技术文档、精读内容 |

## 四、技术架构

### 4.1 组件架构
```
┌─────────────────────────────────────┐
│  CLI (kb command)                   │
│  - kb <url>                         │
│  - kb batch <urls>                  │
│  - OpenClaw Skill                   │
└─────────────────────────────────────┘
           ↓
┌─────────────────────────────────────┐
│  智能路由 (router.py)                │
│  - URL 类型识别                      │
│  - 文件类型识别                      │
│  - 批量处理分发                      │
└─────────────────────────────────────┘
           ↓
┌─────────┬──────────┬──────────┬─────┐
│文章管线  │ OCR 管线  │ 视频管线  │ ... │
│ article │ ocr     │ video   │     │
└─────────┴──────────┴──────────┴─────┘
           ↓
┌─────────────────────────────────────┐
│  LLM 旁注生成 (llm.py)               │
│  - OpenRouter API                   │
│  - OpenClaw 复用                     │
└─────────────────────────────────────┘
           ↓
┌─────────────────────────────────────┐
│  输出格式化 (output.py)              │
│  - Markdown + YAML frontmatter      │
│  - 飞书文档同步 (feishu_sync.py)     │
└─────────────────────────────────────┘
```

### 4.2 技术选型
| 组件 | 选型 | 理由 |
|-----|------|-----|
| CLI 框架 | Typer | 类型安全，自动生成帮助 |
| 文章提取 | trafilatura | 最快最准，内置去噪 |
| OCR 引擎 | RapidOCR | API 稳定，中文优化 |
| 视频处理 | yt-dlp + Whisper | 字幕优先，音频兜底 |
| LLM | OpenRouter / OpenClaw | 多模型，按量付费 |
| 输出格式 | Markdown + YAML | 兼容所有笔记工具 |

## 五、竞品对比

| 维度 | kb-tool | MarkItDown | Crawl4AI | 百度网盘 AI 笔记 |
|-----|---------|-----------|---------|--------------|
| 智能路由 | ✅ | ❌ | ❌ | ❌ |
| 原文 + 旁注 | ✅ | ❌ | ❌ | ⚠️ |
| B 站/抖音 | ✅ | ❌ | ❌ | ❌ |
| 可编程 | ✅ | ✅ | ✅ | ❌ |

1. **智能路由**: 自动识别输入类型，用户无需指定

3. **多平台支持**: B 站/抖音/微信全覆盖
4. **开源可编程**: CLI + API 双模式

## 八、风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|-----|------|-----|---------|
| 微信反爬升级 | 文章无法提取 | 中 | Playwright 兜底 + 镜像站备用 |
| B 站接口变更 | 视频字幕失败 | 中 | Cookies 方案 + 社区维护 |
| LLM API 限流 | 旁注生成失败 | 低 | 多模型切换 + 本地缓存 |
| OCR 精度不足 | 识别错误 | 低 | LLM 校错 + 人工反馈循环 |
